2. Credentials & Tokens to Collect BEFORE Coding
Do not start Claude Code sessions until you have gathered everything in this section. Create a local .env.local file in each app — never commit these to GitHub.
2a. Supabase (Required by: Jaki, Timekeeper/Caregiver)
Go to https://app.supabase.com → your project → Settings → API


SUPABASE_URL=https://xxxx.supabase.co


SUPABASE_ANON_KEY=eyJ...          # "anon public" key


SUPABASE_SERVICE_ROLE_KEY=eyJ...  # "service_role" key (server-side only, never expose client-side)


Also needed from Supabase → Settings → Database:


DATABASE_URL=postgresql://postgres:[password]@db.xxxx.supabase.co:5432/postgres


Check that the following tables exist in your Supabase project (create them if not):


* users — caregiver and patient accounts
* events — watch firmware events
* locations — Arthur's real-time location
* notifications — SOS and push notification log
2b. Firebase / FCM — Push Notifications (Required by: Jaki, Timekeeper/Caregiver)
Go to https://console.firebase.google.com → your project → Project Settings → Cloud Messaging


FIREBASE_PROJECT_ID=your-project-id


FIREBASE_CLIENT_EMAIL=firebase-adminsdk-xxxx@your-project.iam.gserviceaccount.com


FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"


FCM_SERVER_KEY=AAAA...   # Legacy server key (for Capacitor push plugin)


Download google-services.json (Android) and GoogleService-Info.plist (iOS) from: Firebase Console → Project Settings → Your Apps → Download config files


Place them:


* jaki/android/app/google-services.json
* jaki/ios/App/App/GoogleService-Info.plist
2c. Google Maps / Mapbox — Real Map in Jaki (Required by: Jaki)
Option A — Google Maps: Go to https://console.cloud.google.com → APIs & Services → Credentials → Create API Key Enable: Maps SDK for Android, Maps SDK for iOS, Maps JavaScript API


GOOGLE_MAPS_API_KEY=AIza...


Add to jaki/android/app/src/main/AndroidManifest.xml:


<meta-data android:name="com.google.android.geo.API_KEY" android:value="@string/google_maps_key"/>


Option B — Mapbox: Go to https://account.mapbox.com → Access Tokens


MAPBOX_PUBLIC_TOKEN=pk.eyJ1...


MAPBOX_SECRET_TOKEN=sk.eyJ1...   # For downloads only, never ship in app
2d. Apple Developer / iOS (Required if building for iOS)
* Apple Developer account: https://developer.apple.com
* Bundle ID for Jaki: com.yourorg.jaki
* Bundle ID for Timekeeper: com.yourorg.timekeeper
* Generate provisioning profiles and certificates in Xcode → Signing & Capabilities
* Add push notification entitlement: Xcode → Target → Signing & Capabilities → + Push Notifications


APPLE_TEAM_ID=XXXXXXXXXX   # 10-character Team ID from developer.apple.com
2e. Android Keystore — Signing for Release Builds (Required by: Jaki, Timekeeper)
Generate a keystore (run once, store the file safely — losing it means you can't update the app):


keytool -genkey -v -keystore release.keystore -alias hackcessible -keyalg RSA -keysize 2048 -validity 10000


ANDROID_KEYSTORE_PATH=./release.keystore


ANDROID_KEY_ALIAS=hackcessible


ANDROID_KEYSTORE_PASSWORD=your-password


ANDROID_KEY_PASSWORD=your-password
2f. npm Private Registry (Required if publishing shared packages)
If you decide to publish packages/schema and packages/supabase-client as private npm packages:


npm login --registry=https://registry.npmjs.org


NPM_TOKEN=npm_...   # From npmjs.com → Access Tokens → Generate New Token → Automation


Add to packages/schema/.npmrc and packages/supabase-client/.npmrc:


//registry.npmjs.org/:_authToken=${NPM_TOKEN}
2g. Electron Code Signing — Laptop Monitor (Required for Mac/Windows distribution)
Mac (notarization): Go to https://appleid.apple.com → App-Specific Passwords → Generate


APPLE_ID=your@email.com


APPLE_APP_SPECIFIC_PASSWORD=xxxx-xxxx-xxxx-xxxx


APPLE_TEAM_ID=XXXXXXXXXX


Windows (optional for hackathon): Requires a paid EV code signing certificate from DigiCert or Sectigo. Skip for demo — only needed for production distribution.

3. Android Studio Setup
Complete this before running any Capacitor/Android builds.
3a. Install Android Studio
Download from: https://developer.android.com/studio Install the following during setup (or via SDK Manager afterwards):


* Android SDK (API Level 33 — Android 13, minimum target)
* Android SDK Build-Tools 33.0.2+
* Android Emulator
* Android SDK Platform-Tools
3b. Set Environment Variables
Add to your ~/.bashrc or ~/.zshrc:


export ANDROID_HOME=$HOME/Android/Sdk        # Linux


# export ANDROID_HOME=$HOME/Library/Android/sdk  # Mac


export PATH=$PATH:$ANDROID_HOME/emulator


export PATH=$PATH:$ANDROID_HOME/platform-tools


export PATH=$PATH:$ANDROID_HOME/tools


Then reload:


source ~/.bashrc


Verify:


adb --version
3c. Capacitor Android Sync
After Android Studio is installed, run these from inside each app (jaki, caregiver):


npm install


npx cap add android        # Only if android/ folder does not exist yet


npx cap sync android       # Run this every time after npm install or code changes


npx cap open android       # Opens Android Studio for the project
3d. Create an Emulator (AVD)
In Android Studio: Tools → Device Manager → Create Device


* Hardware: Pixel 6
* System Image: API 33 (Tiramisu) — download if not present
* Name: Hackcessible_Pixel6


Or via command line:


avdmanager create avd -n Hackcessible_Pixel6 -k "system-images;android-33;google_apis;x86_64"


emulator -avd Hackcessible_Pixel6
3e. Run on Emulator
npx cap run android       # From inside jaki/ or apps/caregiver/

7. Token Optimization Rules for Claude Code
Follow these rules to stretch your remaining Claude token budget:


1. One app per session. Never run Claude Code across multiple apps in the same session.
2. Give exact file paths. Always tell Claude which file to edit: "Edit apps/caregiver/src/lib/supabase.ts" — this avoids file-search token burn.
3. Run /compact every ~30 messages. This compresses conversation history and drops resolved context.
4. Use Gemini for config and boilerplate. Anything that is "generate a standard config file" should go to Gemini Pro, not Claude Code.
5. Paste errors directly. When something breaks, paste the exact terminal error — don't describe it. Claude resolves exact errors in fewer turns.
6. Batch related tasks. Give Claude 2–3 related tasks in one message rather than one at a time. Example: "Remove isMock, connect real Supabase credentials, and remove the demo banner — all in one pass."
7. End sessions cleanly. When a task is done, end the session (/exit) before starting the next app. Do not let sessions idle open.
