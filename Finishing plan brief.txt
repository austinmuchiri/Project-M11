## Session B ‚Äî Watch Simulator: Write Events to Supabase

**Start from:** `cd timekeeper/apps/watch-simulator && claude`

### B1 ‚Äî Add Supabase client to simulator (simulator.jsx)

File: `apps/watch-simulator/simulator.jsx`

The simulator loads React from CDN. Add the Supabase client the same way:

```html

```

In `simulator.jsx`, initialise the client at the top:

```js
const SUPABASE_URL = 'YOUR_URL';         // hardcode for demo
const SUPABASE_ANON_KEY = 'YOUR_KEY';    // hardcode for demo
const sb = supabase.createClient(SUPABASE_URL, SUPABASE_ANON_KEY);
const DEMO_KID_ID = 'YOUR_KID_UUID';     // the UUID from your kids table
```

**Never commit these credentials** ‚Äî the simulator is not public-facing.

### B2 ‚Äî Fire events on user actions (simulator.jsx)

The simulator has a task start/complete flow. Hook into those UI interactions to write
`task_events` rows:

```js
async function fireEvent(taskId, routineId, status) {
  await sb.from('task_events').insert({
    kid_id: DEMO_KID_ID,
    routine_id: routineId,
    task_id: taskId,
    status,          // 'started' | 'done' | 'missed' | 'paused'
    ts: Date.now(),
  });
}
```

Call `fireEvent` when:
- User taps "Start" on a task ‚Üí `status: 'started'`
- User taps the checkmark / "Done" ‚Üí `status: 'done'`
- Timer expires without action ‚Üí `status: 'missed'`

### B3 ‚Äî Push fake device heartbeat (simulator.jsx)

The caregiver Home screen shows watch battery and last-seen time from the `devices` table.
Add a periodic upsert from the simulator:

```js

    kid_id: DEMO_KID_ID,
    kind: 'watch',

    battery,
    last_seen: Date.now(),
    fw_version: 'sim-1.0',
```

### C6 ‚Äî Offline queue with electron-store (main.ts)

Install: `npm install electron-store`.

Wrap `pushHeartbeat` in a try/catch. On failure, push to a local queue:

```ts
import Store from 'electron-store';
const queue = new Store({ name: 'offline-queue' });

async function safeHeartbeat(h: LaptopHeartbeat) {
  try {
    await client.pushHeartbeat(h);
    flushQueue();
  } catch {
    const q = queue.get('heartbeats', []) as LaptopHeartbeat[];
    queue.set('heartbeats', [...q, h]);
  }
}

async function flushQueue() {
  const q = queue.get('heartbeats', []) as LaptopHeartbeat[];
  if (!q.length) return;
  for (const h of q) await client.pushHeartbeat(h);
  queue.set('heartbeats', []);
}
```

Add an online/offline indicator to the tray popup HTML:
- Green dot + "Connected" when last heartbeat succeeded
- Red dot + "Offline ‚Äî queued N events" when catch fires

### D1 ‚Äî BLE GATT characteristic for events

Confirm there is a writable GATT characteristic for the laptop to read task events.
The expected format is a JSON string or a compact binary struct:

```c
// Minimum viable: JSON string written to characteristic UUID 0xAB01

```

### D2 ‚Äî Laptop monitor BLE reader (new file: src/ble-bridge.ts)

Install: `npm install @abandonware/noble` (Node BLE library for Electron).

Create `src/ble-bridge.ts`:

```ts
import noble from '@abandonware/noble';

const SERVICE_UUID = 'ab00';
const EVENT_CHAR   = 'ab01';
const NUDGE_CHAR   = 'ab02';   // writable ‚Äî laptop pushes nudges to watch

export function startBleBridge(onEvent: (ev: TaskEvent) => void) {
  noble.startScanning([SERVICE_UUID]);
  noble.on('discover', (peripheral) => {
    peripheral.connect(() => {
      peripheral.discoverServices([SERVICE_UUID], (_, services) => {
        services[0].discoverCharacteristics([], (_, chars) => {
          const eventChar = chars.find(c => c.uuid === EVENT_CHAR);
          eventChar?.subscribe();
          eventChar?.on('data', (buf) => {
            const ev = JSON.parse(buf.toString()) as TaskEvent;
            onEvent(ev);
          });
        });
      });
    });
  });
}
```

In `client.ts`, call `startBleBridge` and pipe events to `client.recordEvent`.
